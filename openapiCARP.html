Register the authorisation of ITMOs from the registry identified by regId.

Register authorised ITMOs, from the registry identified by regId.
This enables ITMOs for interoperability by to be issued via a Issuance Action Proposal from the registry.
Note that this operation does not create ITMOs, in creates an ITMO Authorization,
containing an authorized C02 quantity, a cooperative approach id, and an authorization id.
When the ITMOs are subsequently issued by the registry, it must be consistent with this information.
If the ITMO's cooperative approach id is not found, or is not yet authorised, the operation fails.

Authorizations:
basicAuth
path Parameters
regId
required
string (RegistryId) ^[A-Z]{3}\d{2}$
Example: DEU00

The registry identifier of the participating Party registry in which the contact is to be inserted.

Request Body schema: application/json
required

The ITMOs to be registered.

required
object (Authorization)

Abstract superclass for authorisations

cooperativeApproachId
required
string

Cooperative approaches under Article 6.2.

The format of the string is "CANNNN" where the "CA" is the fixed prefix, and NNNN is a four-digit integer,

that shall be incremented by one for each additional agreed cooperative approach, and added to this common nomenclature.

purposeForAuthorization
required
integer [ 1 .. 7 ]

Purpose for authorisation as described by Decision 6/CMA.4 para 17 (j)

Refer to common nomenclatures for code lookup.

OIMPsAuthorized
Array of strings

Only applicable if the authorization is for OIMP.

This field specifies the other international mitigation purpose (OIMP) authorised for this cooperative approaches

quantityCO2
required
integer <int32> [ 1 .. 999999999 ]

Quantity of ITMOs in metric tons of CO2 equivalent.

vintages
required
Array of integers[ items [ 2021 .. 9999 ] ]

Years in which mitigation outcomes occured as covered by the authorisation

authorizationRegistryId
any

The registry the participating Party has access to, for tracking and reccording ITMOs.

underlyingCooperativeApproaches
Array of any

Details, including registries and frameworks used by the ITMOs.

required

The registry identifier of the participating Party registry that authorised the cooperative approach.

caId
required
string
Example: CA0002

The id of the cooperative approach be updated.

Request Body schema: application/json
required

The cooperative approach authorisation to be registered.

cooperativeApproachId
required
string

Cooperative approaches under Article 6.2.

The format of the string is "CANNNN" where the "CA" is the fixed prefix, and NNNN is a four-digit integer,

that shall be incremented by one for each additional agreed cooperative approach, and added to this common nomenclature.

authorisationId
required
string

Authorization Id as assigned by the Party authorizing this cooperative approach.

otherParticipatingParties
required
Array of strings

Identifiers of other participating Parties covered by authorization of cooperative approach.

startDate
required
string <date>

start date of the cooperative approach.

endDate
required
string <date>

end date of the cooperative approach.

firstTransferForOIMP
integer [ 2 .. 4 ]

Only for authorisations for use toward OIMP.

Specifies the first transfer definition used by the Pary pursuant to decision 2/CMA.3, annex, para. 2(b).

Refer to common nomenclatures for code lookup.

metric
required
string

The metric used to measure the ITMO.

As defined per common nomenclature - recognised values are "GHG" and "non-GHG".

Array of objects (NonGHGMetric)

The nonGHG metrics used by this cooperative approach

sectors
required
Array of strings

One or more sectors where the mitigation outcome will occur as per common nomenclatures based on IPCC guidelines.

Refer to common nomenclature for recognised values for "sectors".

mitigationTypes
required
integer

The mitigation type(s) covered by the cooperative approach.

Refer to common nomenclatures for recognised values for "mitigationTypes".

activityTypes
required
Array of integers

The mitigation activity type(s) from which the ITMOs were generated.

Refer to common nomenclature for recognised values for "activityTypes".

Array of objects (ActivityRef)

More information of the activities

authorisedEntityIds
Array of strings

Unique identifier of entities that are authorised as per common nomenclatures,

for this cooperative approach, if applicable.

mitigationOutcomeDocHashes
Array of strings non-empty

(required by Decision 6/CMA.4, annex I, para.4). ITMOs must be traceable to their mitigation outcomes.

This array contains the SHA256 hashes of all mitigation outcome documents for this cooperativeApproach.

The UNFCCC will store all MO documents, and provide an API getMitigationOutcomeDoc(cooperativeApproachId, sha256) that will return the URL of the MO document corresponding to the hash.

otherReferences
Array of strings

Identification of, or coress-reference to, underlying regulations,

frameworks, standards, or procedures,

including any specific methodologies underpinning the cooperative approach.
