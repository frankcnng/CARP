Introduction

This API is part of the communication standards for transactions with Internationally Transferred Mitigation Outcomes (ITMOs) pursuant to decision 6/CMA.4, para. 32.

Participating Parties opting to apply the guidance referred to in decision 6/CMA.4, annex I, chapter I.B are expected to adhere to this API when they interoperate.

Action

Manage actions of participating Party registries

Propose a new action, from the registry identified by regId.<br> If the proposed action is valid, it will be added to the pending actions list,<br> avalable for appropriate registries to process.<br> An actionId is returned by the operation, so regitries can check and augment the status of an action.

Propose a new action, from the registry identified by regId.
If the proposed action is valid, it will be added to the pending actions list,
avalable for appropriate registries to process.
An actionId is returned by the operation, so registries can check and augment the status of an action. If the proposed action is not valid, the operation will fail. The incoming payload is the proposedAction, that also include the ITMOs involved in the action. These will be locked for the duration of the action.

Authorizations:
basicAuth

and two digit registry serial number.

required

Retrieve the actions that are pending processing by the registry identified by regId

Retrieve the actions that are pending processing by the registry identified by regId.

This operation is used by a registry to process actions that have not yet been finalized,

and where the registry is expected to undertake the relevant steps to progress the action.

* If it is a sending registry, complete or terminate the action.

* If it is a receiving registry, accept or reject the action.

This operation can also be used by a registry to query the status of a pending action it is already aware of

* If it is a sending registry, at any point before its finalization.

* If it is a receiving registry, after it has been validated and before its finalization.

The list of actions returned is sorted in descending order or the proposed action data time (ie, the oldest action is first).

Another operation (getfinalizedActions) returns actions with finalized statuses.
