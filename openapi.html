Introduction

This API is part of the communication standards for transactions with Internationally Transferred Mitigation Outcomes (ITMOs) pursuant to decision 6/CMA.4, para. 32.

Participating Parties opting to apply the guidance referred to in decision 6/CMA.4, annex I, chapter I.B are expected to adhere to this API when they interoperate.

Current status

This API is currently in development.

Action

Manage actions of Participating Party Registry Sections

Propose a new action, from the registry identified by regId If the proposed action is valid, it will be added to the pending actions list, avalable for appropriate registries to process. An actionId is returned by the operation, so regitries can check and augment the status of an action.

Propose a new action, from the registry identified by regId If the proposed action is valid, it will be added to the pending actions list, avalable for appropriate registries to process. An actionId is returned by the operation, so regitries can check and augment the status of an action. If the proposed action is not valid, the operation will fail. The incoming payload is the proposedAction, that also include the ITMOs involved in the action. These will be locked for the duration of the action.

basicAuth

required
string^[A-Z]{3}\d{2}$

The registry identifier of the participating Party registry section in which the contact is to be inserted.

required

The proposed action.

required
string

Important note. When a transaction is proposed, this must be the caller registry.

fromAccount
required
integer <int32> [ 1 .. 999999999 ]
toRegistry
required
string
toAccount
required
integer <int32> [ 1 .. 999999999 ]
actionType
required
integer

The action type code. See Common Nomenclatures for action type codes, and their corresponding values and descriptions.

Retrieve the transactions for the registry identified by regId and other parameters

Retrieve the transactions for the registry identified by regId and other parameters. Actions where the registry is both the from registry and the to registry will be retrieved. This API is for management and testing, as it is more complicated than getPendingActions()

Authorizations:
basicAuth
path Parameters
regId
required
string^[A-Z]{3}\d{2}$
Example: DEU00

The identifier of the registry for which actions are to be retrieved. Actions where the registry is both the from registry and the to registry will be retrieved.

query Parameters
offset
integer <int32> [ 0 .. 999999999 ]

The number of items to skip before starting to collect the transactions

limit
integer <int32> [ 1 .. 100 ]

The number of transactions to return

status
string
Enum: "PROPOSED" "ACCEPTED" "REJECTED" "CANCELLED"

Allows to filter by transaction status

from
string <date-time>

The minimum date-time of transactions to return

to
string <date-time>

The maximum date-time of transactions to return

Responses

Production server (uses live data !)

/TBD/action/{regId}/selectedActions
